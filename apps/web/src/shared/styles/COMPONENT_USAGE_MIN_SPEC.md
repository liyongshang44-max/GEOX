# 组件使用最小规范（v1）

> 适用范围：`apps/web/src/views/*`、`apps/web/src/features/**/pages/*`。

## 1) SectionCard 何时使用

使用 `SectionCard` 的条件（满足任一）：

- 一个模块需要同时包含「标题 + 内容体」。
- 同一页面存在两个以上并列信息块，需要统一边框/阴影/间距。
- 模块内部承载列表、指标、步骤、告警摘要等可复用结构。

不建议使用 `SectionCard` 的场景：

- 仅一行即时提示（优先 `StatusPill` 或 `badge`）。
- 紧贴表单字段的局部说明（优先 helper text）。

## 2) EmptyGuide 何时使用

使用 `EmptyGuide` 的条件（需同时满足）：

- 当前区块没有可展示数据（空列表/空时间窗/首次接入）。
- 该空态有明确下一步动作（必须给 `actionLabel + actionTo`）。
- 希望让用户在当前页面内完成恢复路径，而不是跳转错误页。

不应使用 `EmptyGuide` 的场景：

- 请求失败或权限不足（应显示错误态组件，而非空态引导）。
- 数据加载中（应显示 skeleton / loading）。

## 3) partial warning 何时显示

`partial warning` 用于「部分成功、但整体未阻断」的状态，例如：

- 批量任务部分完成，仍有条目待重试。
- 聚合指标存在局部缺失，但主流程继续运行。
- 数据接入成功但质量检查未全部通过。

展示规则：

- 默认使用 warning（黄/橙）色阶，不使用 danger（红）色阶。
- 文案必须指出“缺失范围 + 建议动作”，例如：
  - `部分地块暂缺回传，建议 10 分钟后重试。`
- 若升级到阻断态（例如核心流程失败、人工介入必须立即执行），才允许改为 danger。

## 4) 告警红（danger）使用范围约束

danger/red 仅用于：

- 执行失败（`failed`）
- 明确阻断（`blocked` 且需立即处理）
- 安全/合规拒绝态

danger/red 禁止用于：

- 普通提醒
- 轻微风险提示
- 信息缺失但不阻断的提示（应使用 warning 或 info）
