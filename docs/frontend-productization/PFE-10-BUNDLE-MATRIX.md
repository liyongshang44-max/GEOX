# PFE-10 Bundle Matrix

## 0. Matrix status

This matrix defines the bundle budget review dimensions for Formal Product Frontend v1. Actual values are generated by `CHECK_PFE_10_WEB_BUNDLE_BUDGET.cjs` after `pnpm run build:web`.

| surface | entry / artifact class | budget dimension | baseline value | budget value | status | known risk | later owner |
|---|---|---|---:|---:|---|---|---|
| web total | JS assets | totalJsRawBytes | measured by checker | 2500000 | gated | lazy boundary drift | PFE-11 |
| web total | JS assets | totalJsGzipBytes | measured by checker | 850000 | gated | accumulated route chunks | PFE-11 |
| largest chunk | JS asset | largestJsRawBytes | measured by checker | 1200000 | gated | large shared runtime chunk | PFE-11 |
| largest chunk | JS asset | largestJsGzipBytes | measured by checker | 380000 | gated | large shared runtime chunk | PFE-11 |
| web total | CSS assets | totalCssRawBytes | measured by checker | 550000 | gated | shared style growth | PFE-11 |
| web total | CSS assets | totalCssGzipBytes | measured by checker | 120000 | gated | shared style growth | PFE-11 |
| largest stylesheet | CSS asset | largestCssRawBytes | measured by checker | 350000 | gated | global stylesheet growth | PFE-11 |
| largest stylesheet | CSS asset | largestCssGzipBytes | measured by checker | 80000 | gated | global stylesheet growth | PFE-11 |
| asset inventory | JS count | maxJsAssetCount | measured by checker | 80 | gated | route split churn | PFE-11 |
| asset inventory | CSS count | maxCssAssetCount | measured by checker | 20 | gated | page style proliferation | PFE-11 |
| asset inventory | total count | maxTotalAssetCount | measured by checker | 140 | gated | output artifact sprawl | PFE-11 |

## 1. Interpretation

A budget pass means the current build output remains inside the documented PFE-10 envelope. It does not mean Lighthouse certification, production monitoring, browser matrix completion, or device matrix completion.

## 2. Review requirement

If a future PR must adjust a budget, the PR must include an issue-register entry explaining the reason, the measured baseline, and the later owner. Silent budget weakening is not accepted.
