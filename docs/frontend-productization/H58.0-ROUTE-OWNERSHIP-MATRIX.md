## 0. 目的

本文冻结 H58 frontend productization 的路由所有权矩阵。

本文不声明所有目标路由已经实现。本文只定义：

```text
谁拥有当前 route
谁拥有未来 product route
哪些 route 只能保留 URL-only
哪些 route 必须先验收再迁移
哪些 route 不得进入正式导航
```

---

## 1. 所有权原则

H58 使用以下 route ownership 原则：

1. 一个产品面只能有一个正式 shell owner。
2. route table 拥有页面选择权，layout 不应偷偷替换 children。
3. legacy route 可以存在，但不得进入正式导航。
4. `/app/*` 是目标 canonical family，不得在 broad wildcard 未验收前直接接管 root router。
5. `/operator/twin/*`、旧 `/operator/*`、Customer views、Admin pages 不得先删。
6. product navigation 只指向正式产品概念，不指向工程编号页面。
7. route ownership 的变更必须配套 acceptance。

## 4. Legacy route policy

Legacy route policy is fixed as:

```text
LEGACY_VISIBLE_BY_URL_ONLY
```

until all of the following are true:

1. Product replacement route exists.
2. Product replacement route has acceptance coverage.
3. Old route is absent from formal navigation.
4. Reverse reference check passes.
5. No acceptance depends on the old route.
6. User confirms redirect or deletion.
