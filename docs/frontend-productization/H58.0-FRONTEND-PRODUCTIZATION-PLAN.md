# H58.0 前端产品化计划

Status: H58.0 BASELINE PLAN  
Language: zh-CN  
Scope: Frontend Productization / Runtime Console v1 / Route Ownership / Product IA  
Repo basis: main branch after H52 frontend reset, H52.1 Operator Evidence Twin IA, P57 full runtime freeze

## 0. 目的

本文冻结 GEOX 前端从“研究阶段页面集合”进入“正式产品前端”的产品化计划。

本阶段不重写页面，不删除旧页面，不新增写入口，不改变 backend contract。它只冻结以下内容：

1. 主导航。
2. 路由所有权。
3. 产品视图层级。
4. shell / layout 归属。
5. 旧页面能力迁移策略。
6. 内部治理对象裸露表达的替换方向。
7. replay-backed / live-device nonclaim 的正式前端展示责任。

---

## 1. 阶段命名

本阶段定义为：

```text
H58 Frontend Productization / GEOX Runtime Console v1
```

阶段目标：

```text
把 GEOX 已冻结的 replay-backed production-governed digital twin runtime v1，整理成正式产品前端。
```

本阶段不是：

```text
新增一批 H58 页面
删除 H52 页面
重做 React 应用
打开 AO-ACT 写入口
把 replay demo 宣称成 live production
把 forecast 宣称成 recommendation
把 residual 宣称成 causality proof
把 health acceptance 宣称成 production monitoring
```

## 2. 上游事实基线

H58 必须继承以下上游事实：

### 2.1 H52 route inventory

H52 已确认当前前端混合了三套路由系统：

```text
/customer/* 与 /admin/* 顶层 shell
legacy catch-all Shell + AppShell + SideNav + AppRoutes
注入到 AppRoutes 的 route modules
```

H58 不重新争论这个事实。H58 的任务是把这些路线收敛为产品面。

### 2.2 H52 Operator Evidence Twin

H52 已定义目标入口：

```text
/app/operator
/app/customer
/app/admin
```

并已定义 Operator Evidence Twin / Water Stress Loop 的目标 route：

```text
/app/operator/fields/:fieldId/evidence-twin
/app/operator/fields/:fieldId/evidence-twin/water-stress
```

H58 不得把 `/app/*` 写成“已全面实现入口”，直到对应 shell、redirect、route behavior acceptance 全部通过。

### 2.3 P57 full runtime freeze

P57 已冻结的事实：

```text
full_runtime_mode = replay_backed_production_demo
full_runtime_v1_frozen = true
replay_backed_production_demo_frozen = true
live_device_production_runtime_v1_frozen = false
real_device_deployed = false
live_device_claimed = false
production_gateway_online = false
live_runtime_monitoring_active = false
field_pilot_execution_started = false
ao_act_task_created = false
dispatch_enabled = false
```

H58 前端必须把这些 nonclaims 作为产品界面的一部分。它们不是脚注，不是 debug 字段，也不是仅供 audit 查看。

## 4. Operator Runtime Console

Operator 是主产品面。

正式名称：

```text
Operator Runtime Console
操作员运行控制台
```

一句话定位：

```text
面向操作员的只读数字孪生运行界面，用于查看证据、状态、预测、残差、校准、运行健康和试点准备状态。
```

### 4.1 Operator 一级导航冻结

H58 Operator Runtime Console 的一级导航冻结为：

| 导航 | 路由目标 | 阶段 | 说明 |
| --- | --- | --- | --- |
| Overview | `/operator` or `/app/operator` after route acceptance | H59 | Runtime Overview，总览当前 runtime mode、field status、nonclaim banner、readiness。 |
| Fields | `/operator/fields` or `/app/operator/fields` after route acceptance | H59/H60 | 地块列表，进入单地块 Field Runtime。 |
| Evidence | `/operator/evidence` or `/app/operator/evidence` after route acceptance | H59/H61 | 证据中心，先复用现有 operator evidence 能力。 |
| Forecast | field tab first, console route later | H60/H61 | 预测视图，不是 recommendation。 |
| Calibration | field tab first, console route later | H60/H61 | 校准与 trial plan，不写 model update。 |
| Health | `/operator/health` or `/app/operator/health` after route acceptance | H62 | Runtime Health，展示 P52/P55/P57 的产品化健康状态。 |
| Pilot | `/operator/pilot` or `/app/operator/pilot` after route acceptance | H63 | Pilot Readiness，展示试点准备，不宣称已执行。 |
| Settings | `/operator/settings` or `/app/operator/settings` after route acceptance | H59 | 操作员偏好与边界说明。 |

注意：H52 Operator Evidence Twin IA 中的五个入口仍然成立。H58 的八个 console-level 导航是产品控制台层级；H52 的 Evidence Twin / Water Stress Loop 是 Fields 下的核心产品视图。

### 4.2 Operator 不进入一级导航的旧能力

以下旧能力不得进入 H58 Operator 一级导航：

```text
Twin 总览
Gateway Demo
Operator Workbench
Approvals
Dispatch
Acceptance
ROI Ledger
Field Memory
Judge Config
Sim Config
Dev Tools
Admin Acceptance
```

处理方式：

```text
保留旧 URL
从正式导航撤下
迁移为产品视图 tab / drawer / audit drawer / future module
验收覆盖后再决定 redirect 或 delete candidate
```

```text

```

目标结构：

```text

  Freeze status
  Live-device nonclaim

  Evidence
  State
  Forecast
  Residual
  Calibration

  Audit
```

### 5.1 Field Runtime 链路

产品链路冻结为：

```text
Evidence → State → Forecast → Residual → Calibration → Health → Pilot
```

这条链路替代前端直接暴露如下工程对象：

```text
twin_state_estimate_v1

forecast_residual_v1
calibration_candidate_v1
offline_calibration_trial_plan_v1
runtime_health_report_v1
full_runtime_freeze_packet
```

### 5.2 Field Runtime 禁止事项

Field Runtime 不得：

```text
写 facts
创建 recommendation
提交 approval
创建 AO-ACT task
dispatch
写 ROI
写 Field Memory
把 forecast 说成事实
把 scenario 说成任务
把 residual 说成因果证明
把 health acceptance 说成生产监控
```

## 13. H58 验收边界

H58.0 完成不等于正式前端完成。H58.0 只证明：

```text
产品前端方向已冻结
主导航已冻结
路由所有权已定义
旧页面迁移策略已定义
replay/live nonclaim 已纳入产品契约
没有打开写入口
没有删除旧页面
```

H58.0 不证明：

```text

Operator shell 已改造
Customer override 已清除
Admin shell 已独立
真实设备已接入
production gateway 已在线
field pilot 已开始
```
