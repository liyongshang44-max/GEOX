# GEOX 作业页产品表达冻结文件

## 目的

冻结第四组在作业页上的 customer-facing 表达边界，明确：

- 默认作业页当前主叙事讲什么
- 哪些技术对象当前已后置
- 哪些块属于默认客户视图
- 哪些块当前属于技术附录/专家信息

本文件只冻结当前已验过的前端表达方式，不改变 operation detail 后端主链。

---

## 一、作业页主口径冻结

作业页默认主口径当前冻结为：

- operation detail
- `useOperationDetail`
- `buildOperationDetailViewModel`

说明：

当前作业页客户表达继续以 operation detail 主链为准。

本文件不将 recommendation detail、execution trace 或前端自拼状态定义为主口径。

---

## 二、默认客户视图当前主要讲什么

基于当前已验代码，作业页默认客户视图当前主要围绕以下五类信息组织：

1. 为什么触发
2. 执行到哪了
3. 证据是否完整
4. 验收结果是什么
5. 下一步做什么

当前默认主块主要表现为：

- 作业关键信息
- 执行时间线
- 重点状态提醒
- 执行过程
- 当前闭环摘要
- 证据与验收（整合）
- 价值影响
- 转人工节点（如有）

---

## 三、默认客户视图当前不高权重展示的对象

基于当前已验代码，以下对象当前已不再作为默认主视图中的高权重直接展示内容：

- Task ID
- source
- 规则 ID
- 规则版本
- 原因代码
- 审计附录中的 recommendation / approval / act task 技术编号
- skill trace
- execution trace 相关技术对象

说明：

此处描述的是当前已验页面状态，不等同于对所有未来页面形态的超前承诺。

---

## 四、技术附录当前边界

基于当前已验代码，作业页技术对象当前统一进入：

- `技术附录（默认关闭）`

当前技术附录中可查看的内容包括：

- 来源与解释
- 技能执行诊断
- 执行过程（技术字段）
- 规则与原因代码
- 审计附录

当前冻结口径为：

- 技术对象允许存在
- 当前统一位于技术附录中
- 当前默认关闭
- 当前不散落在主故事流中

---

## 五、技术对象命名冻结

当前已验页面中，技术附录的对外标题已采用产品化/专家化名称，不再直接使用原始工程对象名作为主标题。

当前已冻结示例包括：

- `Skill Trace` → `技能执行诊断`
- `Detail Aside` → `当前闭环摘要`

当前结论仅描述已验过的页面命名状态。

---

## 六、执行过程表达边界

基于当前已验代码，默认客户层中的“执行过程”当前保留：

- 执行模式
- 执行器
- 执行就绪
- 为何转人工

基于当前已验代码，以下内容当前未在默认“执行过程”主块中直接展示：

- Task ID
- execution trace 原始技术字段聚合

这些内容当前位于技术附录中的“执行过程（技术字段）”。

---

## 七、闭环信息表达边界

当前“闭环信息（可折叠）”仍保留在页面中，但当前业务层表达已收紧为较偏客户可读的内容。

当前已验状态下：

- “建议依据（业务）”默认不展开
- 当前主要强调“建议摘要”“为什么触发”

规则 ID、作物阶段、原因代码等技术依据当前位于技术附录中的“规则与原因代码”。

---

## 八、当前闭环摘要冻结

当前 aside 区块已冻结为：

- `当前闭环摘要`

其当前作用是：

- 汇总执行状态
- 汇总验收
- 汇总证据完整性
- 汇总证据缺失项
- 汇总下一步

当前已不再使用英文占位标题。

---

## 九、当前冻结结论

当前第四组作业页表达冻结结论为：

- 默认主视图当前已不再高权重展示 Task ID / source / 规则版本 / 审计附录
- 技术对象当前统一进入“技术附录（默认关闭）”
- 作业页默认主视图当前主要围绕“触发、执行、证据、验收、下一步”组织
- operation detail 当前继续保持客户作业页主口径

后续若作业页默认客户表达发生明显变化，应同步更新本文件。
