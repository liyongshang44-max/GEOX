## 0. 定位

Water Stress Loop v1 是 GEOX 的第一条农业闭环产品线。

它不试图覆盖所有农业问题，不做氮素、病害、经济收益、产量预测、全季节经营优化。

它只证明一件事：

一块大田的水分压力状态，能否从原始信号形成可信判断，并经过建议、审批、执行、验收和灌后验证，形成可追溯闭环。

核心链路：

```text
RawSignal
→ Observation
→ WaterStressState
→ Forecast
→ Scenario
→ RecommendationCandidate
→ Approval
→ OperationPlan
→ AO-ACT
→ AsExecuted
→ Evidence
→ Acceptance
→ WaterResponseVerification
```

产品目标：

1. 看清水分压力来自哪些数据。
2. 判断当前水分状态是否可信。
3. 在证据不足时请求补充观测。
4. 基于当前状态和预测形成灌溉情景。
5. 形成人工可审的灌溉建议候选。
6. 经审批后形成作业计划和 AO-ACT 任务。
7. 接收执行回执和实执记录。
8. 汇总执行证据和验收结果。
9. 比较灌前/灌后状态，验证灌溉是否产生响应。

## 1. 产品边界

### 1.1 本闭环做什么

Water Stress Loop v1 做：

- 土壤水分输入
- 冠层温度输入
- 天气预测输入
- 灌溉事件输入
- 水分状态判断
- 根区土壤水状态
- 根区水分 7 天预测
- 灌溉情景比较
- 灌溉建议候选
- 人工审批链路
- 作业计划
- AO-ACT 任务和回执
- 实执记录
- 执行证据材料
- 验收结果
- 灌后水分响应验证

### 1.2 本闭环不做什么

Water Stress Loop v1 不做：

- 氮素诊断
- 病害诊断
- 杂草诊断
- 经济收益比较
- 全季节产量预测
- 自动审批
- 自动派单
- 自动创建无人农机任务
- 自动写 Field Memory
- 自动写 ROI
- 自动生成客户报告
- 模型自动学习

### 1.3 当前产品原则

早期只证明闭环成立，不证明全自动农业。

所有关键动作保持：

- 人工可审
- 可追溯
- 可解释
- 可回放
- 不确定性可见
- 只在边界允许时进入下一步

## 2. 最小产品链路

用户提出的最小链路：

```text
SoilMoisture + CanopyTemperature + WeatherForecast + IrrigationEvent
→ WaterStressState
→ SenseTask
→ IrrigationRecommendation
→ Post-IrrigationVerification
```

仓库现实中已经具备更长链路，应冻结为：

```text
RawSignal
→ Observation
→ WaterStressState
→ Forecast
→ Scenario
→ RecommendationCandidate
→ Approval
→ OperationPlan
→ AO-ACT
→ AsExecuted
→ Evidence
→ Acceptance
→ WaterResponseVerification
```

其中 SenseTask 不是主线必经节点，而是证据不足时的补证分支：

```text
WaterStressState 缺证 / 冲突 / 不可信
→ ProblemState
→ AO-SENSE VERIFY/WATCH
→ additional Observation
→ re-estimate WaterStressState
```

## 3.4 SenseTask：补充观测任务

### 当前现状

仓库中已有 AO-SENSE：

- ao_sense_v1
- VERIFY
- WATCH

它的定位是：

- 请求额外观测
- 请求验证
- 监控证据缺口
- 不是控制任务
- 不是作业指令
- 不是 AO-ACT

### 当前触发来源

当前 AO-SENSE 主要来自 ProblemState / Judge：

- 数据不足
- 覆盖率不足
- 窗口不支持
- 传感器可疑
- 证据冲突
- 不确定性过高

### Water Stress Loop 中的使用方式

SenseTask 不是主链必经节点。它是补证分支。

触发条件：

- soil moisture sensing window 缺失
- canopy temperature observation 缺失
- weather forecast 缺失
- root-zone state 不可计算
- quality_flags 显示观测可疑
- confidence 低
- evidence_refs 不足
- ProblemState 为 ACTIVE

分支链路：

```text
WaterStressState unavailable / low confidence
→ ProblemState
→ AO-SENSE VERIFY/WATCH
→ additional Observation
→ re-run StateEstimate
```

### 节点职责

SenseTask 回答：

- 当前状态为什么不能判断？
- 需要补充什么观测？
- 需要哪个设备或人工检查？
- 是 VERIFY 还是 WATCH？
- 这个请求是否已经满足？
- 新 observation 是否已经进入后续判断？

## 7. 验收标准

Water Stress Loop v1 前端验收：

1. 页面只展示水分压力闭环，不混入氮素、病害、经济。
2. 能看到 soil moisture 原始信号和标准观测。
3. 能看到 canopy temperature 状态；若未接入主闭环，应显示为辅助信号。
4. 能看到 weather forecast 输入。
5. 能看到 water_state_estimate 或 root_zone_soil_water_state。
6. 能看到 root_zone_soil_water_forecast 7d，如不存在则显示明确缺口。
7. 能看到 irrigation scenario set；若 options 为空，显示数据缺口，不造假。
8. 能看到 recommendation candidate；若只是候选，必须显示 human approval required。
9. 能看到 approval、operation plan、AO-ACT task、receipt、as-executed。
10. 能看到 evidence artifact 和 acceptance result。
11. 能看到 water response verification 的 pre/post state、delta、class transition、verdict、confidence。
12. 页面不创建 task。
13. 页面不自动审批。
14. 页面不派单。
15. 页面不写 ROI。
16. 页面不写 Field Memory。
17. 页面不生成 customer report。
18. 每个节点都能展开查看来源、时间、质量、证据引用和关键 ID。

## 8. 结论

Water Stress Loop v1 是 GEOX 最适合作为第一条农业闭环的产品线。

原因：

1. 输入明确：土壤水分、冠层温度、天气预测、灌溉事件。
2. 状态明确：水分压力、根区水状态、蒸散风险。
3. 动作明确：灌溉。
4. 执行明确：AO-ACT task、receipt、as-executed。
5. 验证明晰：灌前/灌后状态对比。
6. 后端已经有大量现成对象：H31-H45 closure、root-zone state、root-zone forecast、scenario set、recommendation candidate、AO-ACT、acceptance、water response verification。
7. 范围足够小，不会滑向“万能农业平台”。

它应成为 GEOX 的第一条正式产品闭环，即：

```text
猎鹰 1 号
```
