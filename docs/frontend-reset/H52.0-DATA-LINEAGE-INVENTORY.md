## 0. 目标

本文件用于重建 GEOX 前端 Evidence Twin 的数据分层。

目标链路是：

```text
RawSignal → Observation → StateEstimate → Evidence → Verification
```

每一层必须能回答：

1. 数据是什么。
2. 数据从哪里来。
3. 数据什么时候来。
4. 数据质量如何。
5. 被哪个模型或规则使用。
6. 产生了什么状态判断。
7. 判断后来是否被验证。

本文件只做数据谱系归类，不改后端，不改 seed，不改数据库，不删除旧页面。

## 6. Decision / Execution Side Chain：决策执行旁路

该链路不属于 RawSignal → Observation → StateEstimate → Evidence → Verification 的主链，但必须保留，因为它连接“状态判断”与“执行后验证”。

状态：IMPLEMENTED / PARTIAL BY OBJECT

对象：

- root_zone_irrigation_scenario_set_v1
- irrigation_scenario_set_v1
- decision_recommendation_v1
- root_zone_scenario_recommendation_submission
- approval_request_v1
- approval_decision_v1
- operation_plan_v1
- operation_plan_transition_v1
- ao_act_task_v0
- ao_act_receipt_v1
- ao_act_receipt_v1 builder
- as_executed_record_v1
- operation_state_v1
- operation_chain_validator_v1

说明：

这条链回答：

1. 某个 StateEstimate 是否进入情景比较。
2. 情景是否形成 recommendation。
3. recommendation 是否进入审批。
4. 审批是否形成 operation plan。
5. operation plan 是否形成 AO-ACT task。
6. task 是否有 receipt。
7. receipt 是否形成 as-executed。
8. as-executed 是否形成 evidence artifact。
9. evidence artifact 是否形成 acceptance result。
10. acceptance result 是否进入 verification。

前端中它应该作为 StateEstimate 的旁路展开，而不是和 RawSignal → Observation → StateEstimate 主链混在一起。

前端显示名：

```text
决策执行链
```

## 12. H52.0 结论

GEOX 后端已经具备构建 Evidence Twin 的主体材料。

当前问题不是没有数据链，而是：

1. RawSignal、Observation、StateEstimate、Evidence、Verification 分散在不同接口和不同历史阶段。
2. 前端没有统一 view model。
3. 旧前端入口按工程阶段堆叠，而不是按产品主链组织。
4. Verification 目前最完整的是水分响应验证，其他农业验证仍需后续补。
5. Field Memory、ROI、Customer Report 已有后端能力，但不应混入早期 Operator Evidence Twin 主链。

H52 的前端重建应以本数据谱系为基准。
