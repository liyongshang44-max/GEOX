## 0. Purpose

This file inventories the current frontend route surface before H52 frontend reset.

It answers:

1. What routes currently exist.
2. Which files own each route or navigation entry.
3. Which surface each route belongs to: customer, operator, admin, dev, or legacy.
4. Which routes and files are confusing or duplicated.
5. What action should be taken: KEEP, MIGRATE, MODIFY, LEGACY_VISIBLE_BY_URL_ONLY, DELETE_CANDIDATE, or NEEDS_REVIEW.

This file does not change code. It is a planning baseline.

## 1. Action Vocabulary

| Action | Meaning |
| --- | --- |
| KEEP | Keep the route/file as part of the future product surface with minimal changes. |
| MODIFY | Keep the file, but change naming, navigation, layout, or route ownership. |
| MIGRATE | Move the capability into the new H52 route structure. Do not delete immediately. |
| LEGACY_VISIBLE_BY_URL_ONLY | Keep route reachable by direct URL only. Remove from formal nav. |
| DELETE_CANDIDATE | Candidate for deletion after migration, backlink check, and acceptance coverage. |

## 2. Current Router Architecture Findings

### 2.1 Root App Router

Primary file:

- `apps/web/src/app/App.tsx`

Current root behavior:

- `/login` renders login.
- `/`, `/dashboard`, `/dashboard/customer`, `/dashboard/export` redirect into customer dashboard/export.
- `/customer/*` uses `CustomerShell`.
- `/admin/*` uses `AdminShell`.
- all other paths go through `Shell`, which renders `AppShell` and then `AppRoutes`.

Finding:

The router currently mixes three route systems:

1. top-level `/customer/*` and `/admin/*` shells;
2. catch-all legacy `Shell` route using `AppShell`, `SideNav`, `AppRoutes`;
3. route modules injected into `AppRoutes`, many of which duplicate customer/admin/operator routes.

Impact:

This causes route ownership ambiguity. Some route modules still define paths that are now owned by CustomerShell or AdminShell. Some operator pages are split between old `/operator/workbench` routes and new `/operator/twin` routes.

Action:

MODIFY.

H52 should make a new canonical app shell and quarantine old shell routes.
