# H54.0 — Control Connector Preflight

## Purpose

H54.0 is a read-only preflight for connecting the H53.4 water-stress recommendation candidate to the existing control chain.

It does not implement a new approval system, operation-plan system, AO-ACT system, receipt system, evidence system, acceptance system, verification system, ROI ledger, or Field Memory.

## Current upstream chain

The current water-stress derivation chain is:

```text
H53.2 WaterStressState
→ H53.3 Forecast
→ H53.3 ScenarioSet
→ H53.4 RecommendationCandidate
```

The current H53.4 C8 demo recommendation is a `NO_ACTION` candidate:

- `recommendation_id = rec_h53_4_0fa3d7158a6582f5`
- `recommendation_status = RECOMMENDED`
- `selected_scenario_option_id = NO_ACTION`
- `suggested_action.action_type = NO_ACTION`
- `suggested_action.amount_mm = 0`
- `human_approval_required = true`
- `no_direct_execution = true`

## Existing downstream capabilities found in the repository

H54.0 recognizes these existing capabilities as already present:

- recommendation → approval request builder
- recommendation approval decision builder
- operation plan from approved decision builder
- approval request route
- approval decision route
- operation plan route path inside the approval/control route
- AO-ACT control route
- AO-ACT task service

H54 must connect into those capabilities; it must not recreate them.

## Direct compatibility finding

The current H53.4 C8 demo recommendation is not directly compatible with the existing recommendation approval request builder.

The existing builder currently requires a legacy/actionable recommendation shape:

- `status = CANDIDATE`
- `source = ROOT_ZONE_SCENARIO_SELECTION`
- `recommendation_kind = IRRIGATION_CANDIDATE_FROM_SCENARIO`
- non-empty `source_option_id`
- non-empty evidence refs
- `proposed_action.action_type` in `IRRIGATE | DELAYED_IRRIGATION`
- positive `proposed_action.total_irrigation_mm`
- non-negative `proposed_action.total_effective_irrigation_mm`

The current H53.4 C8 demo output is a valid H53.4 recommendation candidate, but it is a `NO_ACTION` recommendation and should not create an approval request, operation plan, or AO-ACT task.

## H54.1 direction

H54.1 should not blindly submit every H53.4 recommendation to approval.

It should introduce a connector/adapter gate with two lanes:

1. `NO_ACTION` lane
   - mark as non-actionable for control-chain creation
   - keep approval, operation plan, AO-ACT, acceptance, and verification pending
   - no writes except a future explicit non-action submission/audit artifact if separately approved

2. actionable irrigation lane
   - only when H53.4 emits a positive irrigation action
   - transform into the existing recommendation approval request builder contract
   - create only `approval_request_v1` in H54.1
   - do not create approval decision, operation plan, AO-ACT task, receipt, acceptance, verification, ROI, or Field Memory

## H54.0 boundary
