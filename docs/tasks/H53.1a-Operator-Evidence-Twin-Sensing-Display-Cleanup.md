# H53.1a — Operator Evidence Twin Sensing Readback Display Cleanup

## Scope

H53.1a is a frontend-only display cleanup on top of the completed H53.1 sensing-only readback chain.

It does not change seed, DB writes, backend derivation, approval, dispatch, AO-ACT task creation, receipt, acceptance, ROI, or Field Memory behavior.

## Task directory

1. Sensing summary card
   - Show that sensing data has been read back.
   - Show that system derivation is still pending.
   - Show that control-chain nodes have not been triggered by this page.

2. Sensing metric cards
   - Display `soil_moisture_percent` as a direct soil moisture metric.
   - Display `coverage_ratio`, `quality_status`, `actual_points`, and `expected_points` as sensing-window quality.
   - Display `rainfall_forecast_mm_72h`, `et0_mm_72h`, and `temperature_max_c_72h` as weather forecast inputs.

3. Pending-stage grouping
   - Group completed input layer separately from pending derivation nodes.
   - Group system derivation nodes separately from human/control-chain nodes.
   - Group execution/acceptance/verification separately from ROI and Field Memory.

4. Boundary preservation
   - Keep the page read-only.
   - Do not add approval, dispatch, AO-ACT task creation, receipt, or acceptance writes.
   - Keep GET-only readback from `/api/v1/operator/fields/:fieldId/evidence-twin`.

5. Acceptance
   - Add static frontend acceptance for the new display contract.
   - Preserve the existing H53.1 sensing readback acceptance.

## Non-goals

- No WaterStressState derivation.
- No Forecast or Scenario derivation.
- No RecommendationCandidate generation.
- No Approval / OperationPlan / AO-ACT connection.
- No AsExecuted / Evidence / Acceptance / Verification writes.
- No ROI or Field Memory writes.
