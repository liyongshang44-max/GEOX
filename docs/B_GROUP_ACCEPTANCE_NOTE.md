## 三、当前仓库实际交付内容

以下内容基于当前仓库实现核查，不基于假设。

---

### 3.1 正式登录入口

当前状态：已存在

实现：

- 路由：`/login`
- 页面：`LoginPage`
- 行为：
  - 输入 access token
  - 调用 `/api/v1/auth/login`
  - 成功后进入 `/dashboard`

确认点：

- 不再要求客户手动写 localStorage
- 登录失败有明确提示（token 缺失 / 无效 / scope 不足 / 服务不可达）

---

### 3.2 会话模型（Session）

当前状态：已建立

实现：

- 前端存在独立 sessionModel
- session 结构包含：

  - token
  - tenant_id
  - project_id
  - group_id
  - role
  - scopes
  - actor_id
  - token_id

确认点：

- session 单一来源：`/api/v1/auth/login`
- 不再使用 demo 默认上下文
- 上下文缺失会阻断进入

---

### 3.3 上下文一致性

当前状态：已收口

实现：

- 登录成功后，tenant/project/group 来自后端
- 浏览器旧 session 与新登录冲突时：

  → 采用新登录上下文覆盖旧上下文

确认点：

- 不存在 silent 混用上下文
- TopBar 显示当前上下文

---

### 3.4 路由门禁（进入控制）

当前状态：已建立

实现：

- `/login` 为唯一入口
- 业务路由通过 `RequireSession` 统一控制

门禁行为：

- 无 token → 跳 `/login`
- token 存在 → 调 `/api/v1/auth/me` 校验
- 校验失败 → 跳 `/login`

确认点：

- 不再允许“直接进入页面再报错”
- 进入系统前有统一校验

---

### 3.5 会话失效与异常处理

当前状态：已建立基础能力

前端已区分：

- AUTH_MISSING
- AUTH_INVALID
- AUTH_REVOKED
- AUTH_SCOPE_DENIED
- AUTH_ROLE_DENIED
- SERVICE_UNAVAILABLE

对应行为：

| 情况 | 行为 |
|------|------|
| token 缺失/失效/撤销 | 清 session + 回 login |
| 权限不足 | 提示“仅允许查看” |
| 服务不可达 | 提示“服务暂不可用” |

确认点：

- 不再全部表现为“token 失效”
- 不再只是控制台报错

---

### 3.6 退出行为（Logout）

当前状态：已实现（本地版本）

实现：

- TopBar 提供“退出（本地会话）”
- 行为：
  - 清除本地 session
  - 跳转 `/login`

确认点：

- 文案明确为“本地会话”
- 未实现 `/api/v1/auth/logout` 联动

---

### 3.7 当前上下文可见性

当前状态：已实现

实现：

TopBar 显示：

- tenant_id / project_id / group_id
- role

确认点：

- 现场可以确认当前试点环境
- 不再“进系统但不知道在哪个环境”

## 六、当前风险与边界

### 6.1 token 分发依赖人工

当前 access token：

- 由实施或技术支持提供
- 没有自助申请或管理机制

风险：

- token 管理需要人工控制

---

### 6.2 logout 非全链路

当前 logout：

- 仅清本地 session
- 不保证服务端 token 失效

风险：

- 不适用于企业级安全要求

---

### 6.3 无用户体系

当前：

- 无用户账号
- 无用户列表
- 无角色管理界面

风险：

- 权限完全依赖 token
