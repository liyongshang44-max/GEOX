# H54.2 — Actionable Irrigation Approval Adapter

## Purpose

H54.2 proves that a future H53.4 positive irrigation recommendation can be adapted into the existing approval request control chain without rebuilding that chain.

H54.2 does not change the current C8 `NO_ACTION` behavior. The current `NO_ACTION` recommendation must still stop at the H54.1 gate.

## Task-line position

```text
H53.2 WaterStressState
→ H53.3 Forecast
→ H53.3 ScenarioSet
→ H53.4 RecommendationCandidate
→ H54.0 Control Connector Preflight
→ H54.1 Recommendation Approval Request Connector Gate
→ H54.2 Actionable Irrigation Approval Adapter
```

## H54.2 scope

H54.2 introduces a small adapter that converts an H53.4-like positive irrigation recommendation into the legacy/actionable recommendation shape already accepted by the existing approval request builder.

The adapter is not a new approval service. It is a translation boundary.

## Adapter input

A future H53.4-like actionable recommendation has this shape:

```text
recommendation_status = RECOMMENDED
selected_scenario_option_id != NO_ACTION
suggested_action.action_type = IRRIGATE | DELAYED_IRRIGATION
suggested_action.amount_mm > 0
human_approval_required = true
no_direct_execution = true
approval_created = false
operation_plan_created = false
task_created = false
```

## Adapter output

The adapter emits the existing approval builder contract:

```text
status = CANDIDATE
source = ROOT_ZONE_SCENARIO_SELECTION
recommendation_kind = IRRIGATION_CANDIDATE_FROM_SCENARIO
source_option_id = selected_scenario_option_id
proposed_action.action_type = IRRIGATE | DELAYED_IRRIGATION
proposed_action.total_irrigation_mm > 0
proposed_action.total_effective_irrigation_mm >= 0
human_approval_required = true
no_direct_execution = true
approval_created = false
operation_plan_created = false
task_created = false
dispatch_created = false
roi_created = false
field_memory_created = false
```

## Runtime proof

The runtime acceptance uses a temporary probe recommendation only. It posts the adapted recommendation to the existing route:

```text
POST /api/v1/operator/recommendations/:recommendation_id/request-approval
```

Expected result:

```text
operator_recommendation_approval_request_submission_v1 created
approval_request_v1 created
approval_decision_v1 not created
operation_plan_v1 not created
ao_act_task_v0 not created
roi_ledger_v1 not created
field_memory_v1 not created
```

## Boundary

Allowed in probe acceptance:

- write temporary probe `decision_recommendation_v1` facts
- call existing approval request route
- create `operator_recommendation_approval_request_submission_v1`
- create `approval_request_v1`
- clean up all probe facts at the end

Forbidden:

- write `approval_decision_v1`
- write `operation_plan_v1`
- write `ao_act_task_v0`
- write `ao_act_receipt_v1`
- write `as_executed_record_v1`
- write `evidence_artifact_v1`
- write `acceptance_result_v1`
- write `water_response_verification_v1`
- write `roi_ledger_v1`
- write `field_memory_v1`
