# H53.2-preflight — Capability and Data Audit Before WaterStressState Derivation

## Purpose

Before implementing H53.2 WaterStressState Derivation, this preflight locks the current repository and local DB facts into an explicit matrix.

It answers three questions:

1. Which input data slots already have table/fact capacity, and which are still only view-model placeholders?
2. Which downstream outputs already have backend capability, and which still need derivation work?
3. Is the H53.1 `c8-sensing-only` lane still clean, meaning it has not written state, scenario, recommendation, approval, AO-ACT, receipt, acceptance, ROI, or Field Memory facts?

## Scope

This task is read-only. It does not create facts, projections, approvals, operation plans, AO-ACT tasks, receipts, evidence artifacts, acceptance results, ROI ledgers, or Field Memory.

## Expected findings

### Input data capacity

- Soil moisture telemetry / observation / sensing window exists and is populated by H53.1.
- Weather forecast input exists and is populated by H53.1.
- Canopy temperature is currently a view-model input slot, not a dedicated scoped source table.
- Irrigation event is currently a view-model input slot; formal operation data should come later from OperationPlan / AO-ACT / Receipt.
- Post-irrigation soil moisture can reuse telemetry / observation / root-zone state structures, but H53.1 has not recorded post-operation observations.
- Execution receipt data is supported by AO-ACT receipt facts, but H53.1 has not recorded receipts.
- AsExecuted / AsApplied, EvidenceArtifact, Acceptance, WaterResponseVerification, ROI, and Field Memory capabilities exist later in the chain, but H53.1 has not generated their rows for `c8-sensing-only`.

### Backend capability

- H53.2 must implement or wire WaterStressState derivation.
- H53.3 must implement or wire Forecast + Scenario derivation.
- H53.4 must implement or wire RecommendationCandidate generation.
- H54 should connect Water Stress recommendations to the existing control plane, not rebuild Approval / OperationPlan / AO-ACT from zero.
