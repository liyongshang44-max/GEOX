# GEOX 首页产品表达冻结文件

## 目的

冻结第四组在首页层面的产品表达边界，明确：

- 哪个页面作为客户/销售主首页
- 哪个页面保留为平台/专家首页
- 客户首页当前允许表达什么
- 客户首页当前不直接高权重表达什么

本文件只冻结当前已验过的表达层角色与边界，不扩展后端事实口径。

---

## 一、首页角色冻结

### 1. 客户主首页

当前已冻结：

- `/dashboard/customer`

作为：

- 客户首页
- 销售演示首页
- 第四组 customer-facing 首页主入口

当前该页围绕以下四类信息组织：

- 地块状态
- 经营汇总
- 待处理事项
- Top 风险地块

对应实现基础见：

- `apps/web/src/views/CustomerDashboardPage.tsx`

---

### 2. 平台/专家首页

当前已冻结：

- `/dashboard`
- `/`

作为：

- 平台运营首页
- 专家/内部使用首页
- 技术运营总控入口

对应实现基础见：

- `apps/web/src/features/dashboard/pages/CommercialDashboardPage.tsx`
- `apps/web/src/app/routes/dashboardRoutes.tsx`

说明：

`CommercialDashboardPage` 当前保留专家属性与技术运营分区，不作为第四组客户首页阻断项。

---

## 二、客户首页当前允许展示的内容

客户首页当前默认允许展示以下 customer-facing 信息：

### 1. 地块状态
例如：

- 总地块数
- 风险地块数
- 严重风险地块数
- 离线地块数

### 2. 经营汇总
例如：

- 未关闭告警
- 待验收数
- 预计成本
- 实际成本

### 3. 待处理事项
例如：

- 总告警
- 未分配
- 处理中
- 已超时
- 今日关闭

### 4. Top 风险地块
例如：

- 地块名称/编号
- 风险等级
- 告警数
- 待验收数
- 最近作业时间

---

## 三、客户首页当前不直接高权重展示的内容

基于当前已验代码，客户首页当前未直接高权重展示：

- internal read model 名称
- `field_sensing_overview_v1`
- `field_fertility_state_v1`
- `/api/v1/...` 接口路径标题
- debug source / trace 对象
- source observation ids
- source devices
- internal skill trace
- execution trace

说明：

此处描述的是当前已验首页页面状态，不等同于对整个前端的全局永久承诺。

---

## 四、首页信息层级冻结

客户首页当前叙事顺序冻结为：

1. 当前整体状态
2. 当前需要处理什么
3. 风险与成本情况
4. 哪些地块最值得关注

当前不将以下内容作为客户首页主叙事：

- 技能总览大屏
- telemetry 总控台
- runtime 总控台
- 控制链路调度页

这些内容若存在，保留在平台/专家首页。

---

## 五、与平台首页的分工冻结

### 客户首页负责
- 让客户一眼看懂当前经营状态
- 让销售稳定演示 GEOX 的业务价值
- 给出风险、待处理、经营结果的直观表达

### 平台/专家首页负责
- 展示技能态势
- 展示运行态势
- 展示读模型、执行队列、专家控制信息
- 服务内部运营与专家诊断

---

## 六、当前冻结结论

当前第四组首页表达冻结结论为：

- `/dashboard/customer` = 客户主首页
- `/dashboard` / `/` = 平台/专家首页
- 客户首页当前已表现为围绕地块状态、经营汇总、待处理事项、Top 风险地块表达
- `CommercialDashboardPage` 当前保留平台/专家属性，不纳入第四组 customer-facing 首页阻断项

后续若首页客户表达发生明显变化，应同步更新本文件。
